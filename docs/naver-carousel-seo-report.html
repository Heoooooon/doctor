SEO Research Report

네이버 캐러셀 구조화 데이터, 서울이건치과에는 이렇게 적용합니다

공식 가이드는 맞지만, 검색 노출을 보장하는 마법 버튼은 아닙니다. 현재 사이트 구조와 의료광고 기준을 함께 보며 적용 범위를 정리했습니다.

조사일 2026-06-30 대상 Naver Search Advisor 권장 보수적 적용
결론 공식 요건 현재 상태 적용안 구현 예시 블로그 검토 체크리스트 출처

결론

네이버 캐러셀 구조화 데이터는 목록형 콘텐츠를 이해시키는 보조 신호입니다. 서울이건치과는 모든 페이지에 넣기보다, 실제 목록 구조가 있는 안내 페이지에만 제한적으로 넣는 편이 안전합니다.

맞는 방향

공식 가이드 기반

네이버 Search Advisor의 캐러셀 문서는 공식 자료입니다. `ItemList`와 `ListItem` 구조를 사용하면 됩니다.

주의

노출 보장은 아님

정상 마크업이 검색 결과 캐러셀 노출, 순위 상승, 클릭률 상승을 보장하지는 않습니다.

추천

첫 적용은 `/board`

현재 사이트에서는 `boardCarouselItems` 6개 안내 카드가 가장 적합하고 위험이 낮은 첫 후보입니다.

실무 판단: 캐러셀 SEO는 주력 전략이 아니라 보조 정리 작업입니다. 우선순위는 `Dentist` 구조화 데이터, sitemap, robots, canonical, FAQPage, 네이버 플레이스 정보 정합성입니다.

네이버 공식 요건

핵심은 목록을 `ItemList`로 묶고, 각 항목을 `ListItem`으로 표현하는 것입니다.

항목 권장 처리 서울이건치과 적용 메모
`@type` 상위는 `ItemList`, 하위는 `ListItem` 홈의 안내 카드 목록에 1개만 추가
`position` 순서가 있는 목록이면 1부터 정수로 명시 현재 배열 순서를 그대로 사용
`image` 깨지지 않는 원본 이미지 URL 사용 로고만 반복하거나 중복 이미지를 쓰지 않기
`url` 상대 경로가 아닌 절대 URL `https://egundc.com/board/...` 형태
목록 품질 너무 적거나 중복된 항목은 피함 6개 안내 카드면 최소 조건은 충족

현재 사이트 상태

기존 SEO 기반은 어느 정도 잡혀 있고, `ItemList`만 아직 없습니다.

이미 있는 구조화 데이터

  • `components/seo/JsonLd.tsx:3`의 `Dentist` JSON-LD
  • `components/seo/JsonLd.tsx:93`의 `FAQPage` JSON-LD
  • `app/board/[slug]/page.tsx:53`의 개별 안내 페이지 `WebPage` JSON-LD

첫 적용 후보

  • `lib/board-carousel.ts:14`의 `boardCarouselItems`
  • `app/page.tsx:36`의 홈 목록 렌더링
  • `app/board/[slug]/page.tsx:10`의 상세 페이지
  • `app/sitemap.ts:29`의 URL과 이미지 등록

권장 적용안

작은 범위에서 정확히 넣고, 검색 결과 변화는 네이버 Search Advisor에서 관찰하는 방식이 좋습니다.

진행

  1. 기존 `Dentist`, sitemap, robots, canonical, FAQPage 품질을 유지합니다.
  2. 홈 또는 안내 목록 영역에 `boardCarouselItems` 기반 `ItemList` JSON-LD를 추가합니다.
  3. Schema.org Validator로 오류를 확인합니다.
  4. 네이버 Search Advisor 수집 요청 후 리포트를 관찰합니다.

보류

  • 환자 후기 목록을 캐러셀 SEO로 밀어내는 방식
  • 치료 전후사진 목록을 검색 노출 목적으로 강조하는 방식
  • 치료 효과, 클릭률, 노출을 보장하는 표현
  • Google에서도 같은 캐러셀 효과가 난다고 안내하는 표현

구현 예시

현재 데이터 구조를 그대로 활용하면 아래처럼 만들 수 있습니다. 

const itemListJsonLd = {
  '@context': 'https://schema.org',
  '@type': 'ItemList',
  itemListElement: boardCarouselItems.map((item, index) => ({
    '@type': 'ListItem',
    position: index + 1,
    item: {
      '@type': 'WebPage',
      name: item.title,
      description: item.description,
      image: absoluteUrl(item.image),
      url: absoluteUrl(item.href),
    },
  })),
}

<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify(itemListJsonLd) }}
/>

블로그와 대행 글을 볼 때 기준

블로그 글은 구현 방향을 잡는 데 도움은 되지만, 공식 문서보다 강하게 말하는 부분은 걸러야 합니다.

맞는 말

  • 네이버 캐러셀은 `ItemList/ListItem` 구조로 표현합니다.
  • JSON-LD가 실무상 가장 다루기 쉽습니다.
  • 이미지와 URL을 정확히 넣는 것이 중요합니다.

조심할 말

  • 넣으면 캐러셀에 노출된다는 표현
  • CTR이 크게 오른다는 단정
  • Google에도 같은 효과가 난다는 표현
  • 치과 후기와 전후사진을 캐러셀로 밀자는 제안

서울이건치과 안전 문구: 검색엔진이 각 안내 페이지의 제목, 이미지, URL을 더 명확히 이해할 수 있도록 보조 데이터를 제공합니다. 검색 결과 노출 방식은 검색엔진 정책에 따라 달라질 수 있습니다.

QA 체크리스트

구현 후에는 아래 항목을 기준으로 확인합니다.

1JSON-LD가 `https://schema.org` 컨텍스트를 사용합니다.
2`ItemList.itemListElement`가 3개 이상이고 중복이 없습니다.
3모든 `url`과 `image`가 `https://egundc.com/...` 절대 URL입니다.
4이미지가 깨지지 않고, 로고나 기본 이미지 반복만으로 구성되지 않습니다.
5JSON-LD 항목이 실제 페이지의 보이는 목록과 일치합니다.
6Schema.org Validator에서 오류가 없습니다.
7네이버 Search Advisor 수집 요청 후 리포트를 관찰합니다.
8노출, 순위, CTR 보장 표현을 내부 문서와 외부 문구에 쓰지 않습니다.
